# 2026 暑假 Model 方向 42 天冲刺手册

## 0. 使用方式
- 这份手册按“**每天固定时间块 + 当天学习资料 + 当天代码产出 + 当天实验记录**”组织。
- 目标不是“看完资料”，而是“每天都留下可核验成果”：Git 提交、实验日志、Notebook、README 更新、指标截图、代码运行结果。
- 对照考核文件，重点覆盖：暑期项目进展、论文复现、开源/工程成果、申请材料、答辩重点。  

## 1. 暑期最终目标
1. 完成 1 个 Kaggle 完整项目，形成可展示的训练/验证/提交闭环。
2. 完成 1 篇论文复现，能解释方法、结构、损失函数、数据处理和结果差异。
3. 完成 1 个 Model 工程项目，具备训练、推理、可视化或 API 能力。
4. 形成完整答辩材料：代码仓库、实验日志、阶段总结、结果文件、PPT、个人贡献说明。

## 2. 固定日程模板
> 每天都按这个节奏执行，避免临时决定做什么。

| 时间 | 内容 | 产出要求 |
|---|---|---|
| 09:00-09:40 | 读资料 / 看教程 / 读论文 | 写 5-10 条笔记 |
| 09:40-11:30 | 核心开发 | 完成当天主要代码 |
| 11:30-12:00 | Git commit + 记录日志 | 至少 1 次 commit |
| 14:00-16:30 | 训练 / 调参 / 跑实验 | 产出结果文件或截图 |
| 16:30-17:30 | 分析结果 | 写实验结论 |
| 19:00-20:00 | 二次阅读 / 补知识点 | 整理思路 |
| 20:00-21:30 | 更新 README / 明日计划 | 更新仓库文档 |
